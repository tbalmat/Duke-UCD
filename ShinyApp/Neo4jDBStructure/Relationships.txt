The following query is undirected and reports x->y relationships as both x, y, r and y, x, t
match (x)-[r]-(y) return distinct labels(x), labels(y), type(r), count(1) order by labels(x), labels(y), type(r)

Use a directed query to report originating and destination node types:
Note that arrows must always be directed into a node (->[r] does not make sense because r is not a node)

match (x)-[r]->(y) return distinct labels(x), labels(y), type(r), count(1) order by labels(x), labels(y), type(r)
